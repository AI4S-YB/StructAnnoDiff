1. 当前结论

算法状态

当前算法已经从原始 gene span 匹配切换为基于子特征的转录本级匹配。 基因对得分来自基因内最佳 before/after 转录本对，分母是单个转录本的 exon 总长度。

主要修复

对已有清楚 1:1 锚点的重叠群，会过滤额外的弱桥接边，避免两个独立基因座因为少量尾部 exon/UTR 重叠被误归为 complex。

结果状态

全部 7 个物种已重新运行；结果表、IGV tracks、跨物种主图和单物种图均已刷新。 验证脚本和单元测试通过。

菠萝区域 contig05:19,767,616-19,774,408 已作为定点检查案例： 当前输出为 g05902 -> lcfv2_06000 的 1:1 exon/UTR 边界调整， 以及 g05903 -> lcfv2_06001 的 1:1 exact，不再被误判为 complex。

2. 计算逻辑

解析 GFF

读取 gene、mRNA/transcript、exon、CDS、UTR。

清洗模型

剔除没有显式 exon 或 CDS 证据的 mRNA；无有效 mRNA 的 gene 不参与 overlap。

补全结构

UTR 只由 exon/CDS 推导；缺 exon 时只用 CDS 和显式 UTR 推断 exon。

转录本打分

同 seqid、同 strand 内，基因对得分取最佳转录本对的 exon overlap。

图解析

候选边构成 before/after 二部图，连通分量归为 1:1、split、merge、complex。

结构分类

只对严格 1:1 基因对评估代表性转录本 exon/CDS/UTR/isoform 变化。

图示强调当前算法的核心：以单个转录本 exon 总长度作为 overlap 分母，先获得最佳转录本对，再在基因二部图中解析基因座事件； 对已被强 1:1 锚点解释的重叠群，低 CDS 支持、低 Jaccard 的额外桥接边会被过滤。

2.1 模型边界

GFF 中的 gene feature span 可能过宽或不可靠，因此当前匹配不使用 raw gene span。 模型的 start/end 由真实子特征推导；原始 gene 边界保存在 before_gene_start/end 和 after_gene_start/end 中，仅用于审计。

2.2 转录本级 overlap 得分

当前关键定义如下。这里的 tx_length 是一个转录本 merged exon 区间的总长度， 不是 raw gene span，也不是一个 gene 内所有 isoform 合并后的长度。 

tx_length(t) =
  sum(length of merged exon intervals in transcript t)

tx_overlap_score(t_before, t_after) =
  overlap(exons(t_before), exons(t_after))
  / min(tx_length(t_before), tx_length(t_after))

gene_pair_score(g_before, g_after) =
  max(tx_overlap_score(t_before, t_after)
      for all before/after transcript pairs)

采用 min() 作为分母后，较短转录本被较长转录本充分包含时会被视为强对应证据。 这符合“被包含就是被包含”的当前判断原则。

2.3 CDS 兼容性救援

如果某一侧没有显式 exon，exon 只能由 CDS/UTR 下限推断。此时同时计算 CDS-to-CDS 的最佳转录本对 overlap。 由于 CDS 本身是 exon 的组成部分，CDS-only 注释不应因为另一侧补充了 UTR 或非编码 exon 而被稀释成未匹配。

2.4 弱桥接边过滤

containment/min-denominator 策略会更容易保留包含式重叠，但在相邻基因存在 UTR 或 exon 尾部接触时， 也可能把两个本来已经能两两对应的基因座连成一个 complex。当前剪枝规则为：

  1. 先在候选图中寻找互为最佳的 1:1 锚点。
  2. 如果一条边连接的 before 和 after 两端都已经各自有不同的强锚点，则认为它可能是桥接边。
  3. 只有当该桥接边 CDS overlap < 0.5 且 feature Jaccard < 0.25 时才剪掉。
  4. 如果桥接边解释了未锚定成员，或 CDS/Jaccard 支持足够，则保留，以免漏掉真实 split/merge/complex。

结果解读

  • F. vesca 的代表性 exon 变化比例仍然很高。当前数据中 1:1 exact 只有 2,835 个，而 1:1 代表性 exon changed 为 19,875 个，说明校准前后存在大范围 exon 边界或 exon 数目变化；这更像全局注释风格差异或大规模校准，而不是多转录本重复计数造成的假高。
  • Pineapple 的 weak bridge pruning 为 228 条，说明此前确实存在不少“两个清楚 1:1 被弱尾部重叠连成 complex”的情况；修复后仍保留 23 个 complex。
  • Rice complex 事件数较高，为 449，需要后续人工抽样检查是否存在真实复杂重排、大片段重复区域或注释模型过密造成的多对多连接。
  • Peach 的 exact 比例最高，代表性 exon 改变比例最低，说明校准前后结构整体较稳定。

5. IGV 和人工校验方式

当前已经为每个物种导出 IGV 友好的 BED、GFF3 和 TSV track。推荐在 IGV 中同时加载： before GFF、after GFF、以及对应的 results/tracks/{Species}_annotation_changes.bed 或 .gff3。

文件用途
results/tracks/*_annotation_changes.bed 适合快速在 IGV 中浏览事件区间，颜色按 syntenic/split/merge/complex/strict unmatched 区分。
results/tracks/*_annotation_changes.gff3 保留事件属性，适合在 feature tooltip 中查看 before/after gene、match type 和 subtype。
results/tracks/*_annotation_changes.tsv 用于按坐标、基因 ID 或事件类型过滤，适合批量抽查。

定点校验示例

这个区域用于验证“已有两个转录本清楚两两对应时，其他弱重叠关系应被过滤”的逻辑。 当前结果符合预期。

8. 注意事项

  • Strict unmatched 不是生物学 novel/deleted。 这些基因只是按当前坐标与结构规则未匹配到另一侧，summary 中字段名保留为 novel_genes 和 deleted_genes 是为了兼容已有结果表。
  • Panel A 更保守。 它统计完全没有坐标重叠的基因座，因此比 strict unmatched 更接近“新增/删除基因座”的概念。
  • 多转录本不会在 Panel B 中重复计数。 split/merge/complex 是 gene/locus 连通分量事件，不是转录本事件。
  • Panel C 只看代表性转录本。 before 取 primary mRNA，after 取最佳匹配 mRNA，避免 after 多转录本展开导致重复计数。
  • CDS phase 被完全忽略。 phase-only 变化不计为结构变化。
  • 仍建议人工抽样。 对 F. vesca 的高比例 exon 变化、Rice 的较多 complex、以及每个物种的高影响区域，应继续用 IGV track 做人工抽查。
