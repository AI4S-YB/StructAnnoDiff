基于当前项目代码的最新逻辑：以基因座坐标互逆重叠建立前后一一对应关系，再对确认的 1:1 基因进行结构变化评估。

默认分析对象

run_locus_comparisons.py 默认使用 --gene-scope mrna，即只比较含 mRNA/transcript 的基因模型。 但 mRNA 必须在输入文件中至少有显式 exon 或 CDS；只有 gene/mRNA/UTR 而没有 exon/CDS 证据的记录会被剔除，不进入 overlap 分析。 ncRNA、tRNA 等非 mRNA 基因类型不进入当前目标图与核心统计，这是为了与本项目的 mRNA 结构校准目标保持一致。

2. 总体流程

解析 GFF

读取 gene、mRNA/transcript、exon、CDS、five_prime_UTR、three_prime_UTR。

补全结构

保留原始 gene 边界用于审计；只从真实子特征推导缺失的 UTR 或 exon，并剔除无 exon/CDS 证据的转录本。

坐标匹配

按 seqid 和 strand 分组，用扫描算法寻找候选基因对；打分取基因内最佳转录本对的 exon footprint overlap。

基因座归类

对候选对构建二部图连通分量，区分 1:1、split、merge、complex、弱重叠未决、严格未匹配 before/after 基因。

结构评估

仅对确认的 1:1 基因进行 exon、CDS、UTR、边界和多转录本变化分类。

流程图强调两条主线：非 1:1 基因座只做事件级统计；确认 1:1 后才进入 exon/CDS/UTR/isoform 结构变化评估。

3. 数据模型与特征补全

基因模型

GeneModel
  gene_id, seqid, start, end, strand
  raw_start, raw_end
  mrnas: list[MRNAModel]

MRNAModel
  mrna_id
  raw_start, raw_end
  exons: list[ExonFeature]
  cds:   list[CDSFeature]
  utrs:  list[UTRFeature]
  has_explicit_exon: bool

start/end 是由 exon/CDS/UTR 子特征得到的模型跨度； raw_start/raw_end 保存 GFF 中原始 gene 行的边界，仅用于审计输出，不参与匹配和结构变化判定。

结构签名

exon_signature = sorted((start, end))
cds_signature  = sorted((start, end))
utr_signature  = sorted((start, end, utr_type))

transcript_signature = (
  exon_signature,
  cds_signature,
  utr_signature
)

gene_structure_signature =
  sorted(transcript_signature for all transcripts)

结构签名不依赖前后文件中的基因 ID 或转录本 ID，适合处理人工校准后 ID 完全改变的情况。

缺失结构的补全策略

情况当前处理目的
mRNA/gene 没有显式 exon，也没有 CDS 该 mRNA 不参与 overlap；若 gene 没有剩余有效 mRNA，则整个 gene 从比较模型中剔除。仅有 UTR 但无 exon/CDS 的转录本同样剔除。 避免错误或未完成的 gene/mRNA 原始 span 制造超长假 overlap、假 split/merge/complex 或假 unresolved。
UTR 行缺失，但有 exon 和 CDS 用 exon 中落在 CDS 区间外的部分推导 5' 或 3' UTR；负链会交换基因组坐标方向上的 5'/3' 判定。 避免仅因输入文件未显式写 UTR 而漏掉 UTR 变化。
UTR 行缺失，且没有 exon 不再用 mRNA 或 gene 原始边界合成 UTR；没有真实 exon 时，只保留 CDS 和显式 UTR 证据。 避免错误的原始 gene/mRNA span 制造假 UTR 或假 exon。
exon 行缺失 用 CDS 与显式 UTR 区间的并集合并为推断 exon，并标记 exons_are_inferred。 CDS 是 exon 的一部分；推断 exon 只是下限证据，不等同于完整 exon 注释。
gene 边界与子特征不一致 模型跨度改用所有 mRNA 子特征的最外侧；原始 gene 边界仍保留。 匹配和边界变化评估都基于真实子特征，原始 gene 边界只作为审计列输出。

4. 基因座匹配算法

校准前后 ID 不可靠，因此匹配只依赖基因组坐标、染色体/scaffold 和链向。当前默认模式是 hybrid：严格 1:1 和 split/merge/complex 构图都先在基因内寻找最佳转录本对， 再用该转录本对的 exon footprint overlap 作为基因对得分。 

tx_length(t) =
  sum(length of merged exon intervals in transcript t)

tx_overlap_score(t_before, t_after) =
  overlap(exons(t_before), exons(t_after))
  / min(tx_length(t_before), tx_length(t_after))

gene_pair_score(g_before, g_after) =
  max(tx_overlap_score(t_before, t_after)
      for all transcript pairs)

关键点

当前代码使用 overlap / min(tx_length_before, tx_length_after)，默认阈值为 0.5。 tx_length 是单个转录本的 exon 总长度，不是 raw gene span，也不是一个基因内所有 isoform 的合并长度。 因此只要较短转录本被较长转录本充分包含，就视为强基因座对应证据。

但当任一侧没有显式 exon、exon 是从 CDS/UTR 推断而来时，算法还会计算 CDS-to-CDS reciprocal overlap。 严格 1:1 证据取 feature reciprocal 与 CDS reciprocal 的较高值。原因是 CDS 本身就是 exon 的一部分， CDS-only 注释不能因为另一侧补充了 UTR/exon 长度而被错误降级为 unresolved。

候选基因对条件

  1. seqid 相同；没有公共 seqid 时该物种比较返回空结果。
  2. strand 相同；反义重叠不会视为同一基因座。
  3. 最佳转录本对的 gene_pair_score >= 0.5，阈值可由 --reciprocal-overlap 修改；缺显式 exon 的 coding 模型允许 CDS 兼容性救援。

搜索实现采用每个 seqid x strand 内按起点排序的双指针扫描。该策略保留所有满足阈值的候选对， 后续再由连通分量决定是 1:1、拆分、合并还是复杂事件。

Containment 图的弱桥接边剪枝

如果一条弱桥接边只是把两个已经各自拥有强 1:1 锚点的相邻基因连接到同一个分量，算法会剪掉这条桥接边。 互为最佳锚点由最佳转录本对 score、feature Jaccard 和 CDS overlap 共同确定。 弱桥接边需要同时满足：两端都有不同的互为最佳锚点、桥接边没有足够 CDS overlap、feature Jaccard 很低。 这用来避免把两个独立 1:1 误升为 complex，同时保留真正的 split/merge/complex。

Strict unmatched 的保守口径

使用最佳转录本对的 overlap / min(tx_length_before, tx_length_after) 后，完整或高比例包含关系会直接进入候选匹配图。 因而 unresolved_overlap_after / unresolved_overlap_before 只保留低于阈值但仍有坐标关联的边缘情况。

连通分量解析

连通分量规模输出类别含义
1 before : 1 aftersyntenic确认一一对应，进入结构变化分类。
1 before : N aftersplit一个原基因被拆分成多个校准后基因。
N before : 1 aftermerge多个原基因被合并成一个校准后基因。
M before : N aftercomplex复杂重排，无法简化为单个拆分或合并。
弱 containment 重叠但未达 reciprocal 阈值unresolved_overlap坐标上存在较强包含式重叠，但不能确认一一对应；不计入严格新增或删除。
0 before : 1 after，且无弱 containment 重叠strict_unmatched_after严格规则下未能匹配到 before 基因的 after 基因；不直接等同于生物学新基因。
1 before : 0 after，且无弱 containment 重叠strict_unmatched_before严格规则下未能匹配到 after 基因的 before 基因；不直接等同于生物学删除基因。

采用连通分量而不是贪心最佳匹配，是为了避免把真实的 split/merge 事件误拆成多个局部 1:1 匹配。 对 complex 分量，change log 和 IGV track 只输出分量内真实存在 overlap edge 的 before-after 对； 不再输出完整的 before × after 笛卡尔组合。

5. 1:1 基因的结构变化分类

结构变化只对 syntenic 1:1 基因对进行详细评估。当前评估同时产生两个层次的结果： 一是互斥的详细 subtype，用于总表闭合；二是非互斥的 one_to_one_* 属性，用于图 2 的结构变化热图。

基础阈值

参数默认值作用
--boundary-tol10 bp原始 gene 起止坐标差异超过该值时，记为 gene boundary changed。
--cds-change-pct0.1CDS 长度相对变化达到 10% 时，记为 CDS 显著变化；CDS 坐标改变也会触发 CDS change。phase 字段不参与结构变化判断。
--utr-change-pct0.1同一侧 UTR 在前后均存在且长度变化达到 10% 时，记为 UTR refined；UTR 坐标变动也会纳入。

非互斥结构属性

属性列判定口径
one_to_one_exact原始 gene 边界未超过容差，且所有转录本结构签名完全一致。
one_to_one_gene_boundary_changed原始 gene 起点或终点变化超过 boundary_tol。
one_to_one_utr_added代表转录本中新出现 5' 或 3' UTR。
one_to_one_utr_lost代表转录本中原有 5' 或 3' UTR 消失。
one_to_one_utr_exon_added代表转录本的 UTR-only exon 数增加。
one_to_one_utr_exon_removed代表转录本的 UTR-only exon 数减少。
one_to_one_utr_refinedUTR 坐标或长度发生变化，但不是简单的 UTR 出现/消失。
one_to_one_coding_exon_gain代表转录本中与 CDS 重叠的 exon 数增加。
one_to_one_coding_exon_loss代表转录本中与 CDS 重叠的 exon 数减少。
one_to_one_exon_boundary_refinedcoding exon 和 UTR-only exon 数均不变，但 exon 坐标发生改变。
one_to_one_cds_changeCDS 长度达到阈值变化，或 CDS 坐标签名改变；phase 字段被完全忽略。

当前关键修正

外显子增减现在分成 coding_exon_count 和 utr_only_exon_count 两条独立路径。 一个新增 UTR-only exon 即使伴随 CDS 边界变化，也不会被误计为 coding exon gain。

互斥 subtype 与汇总表

classify_syntenic_change() 会为每个 1:1 基因生成一个详细 subtype，例如 utr_exon_added、exon_gain_cds_extended、cds_boundary_refined、 isoform_mrna_1x2_restructured。随后 generate_tables.py 按固定优先级聚合到互斥大类： 

Exact
Boundary_refined_only
UTR_added / UTR_lost
UTR_exon_gained / UTR_exon_removed
UTR_refined
Coding_exon_gain / Coding_exon_loss
Exon_boundary_refined

互斥汇总的所有类别之和必须等于 Syntenic。其中 Boundary_refined_only 只表示没有其他结构变化时的纯 gene 边界变化；若 gene 边界变化同时伴随 UTR/CDS/exon/isoform 变化， 该信息由非互斥列 Any_gene_boundary_changed 保留。

6. 多转录本处理

当前方法不是简单丢弃多转录本，而是采用“代表转录本详细比较 + 全基因转录本签名校验”的组合策略。

代表转录本选择

  1. 校准前基因选择 primary_mrna：优先 CDS 总长度最长，其次 exon 数更多，完全相同则按文件读取顺序。
  2. 校准后基因不是盲选最长转录本，而是在所有 after mRNA 中寻找与 before primary 最相似的转录本。
  3. 相似度优先考虑完整结构签名相同，其次 CDS、exon、UTR 签名相同，再比较共同片段数和长度差异。

全基因层面校验

exact 要求所有转录本的结构签名集合完全一致，而不是只要求代表转录本一致。 因此非代表转录本发生变化时，基因不会被误判为 exact。

isoform_change 捕获 mRNA 数量变化、总 exon 数或总 CDS 长度变化，以及非代表转录本结构变化。

方法边界

当前算法没有对每一个 isoform 做完整的前后一对一转录本匹配。代表转录本之外的细节主要归入 isoform_change，不会进一步拆成每个非代表 isoform 的具体 exon/CDS/UTR 子类型。

9. 当前算法边界与仍可优化点

mRNA 范围限定

默认只分析 mRNA/transcript 结构。若未来要比较 ncRNA/tRNA，需要显式切换 --gene-scope all 并重新定义相应的结构统计口径。

多 isoform 细节

当前详细 exon/CDS/UTR 子类型基于代表转录本；非代表转录本变化主要归入 isoform_change。 若需要转录本级别结论，可新增 isoform-to-isoform 全量匹配层。

复杂事件未细分

split、merge、complex 只在基因座事件层面计数，没有进一步给每个子基因对拆解结构变化。 这避免过度解释不稳定的一对多/多对多关系。

解释结果时需要注意

图 2 的结构类别是非互斥属性。同一个 1:1 基因可以同时计入 UTR refined、CDS changed、gene boundary changed 和 isoform changed。因此这些百分比不能相加为 100%；只有 locus_comparison_summary.csv 中的互斥类别可以相加并闭合到 Syntenic。
