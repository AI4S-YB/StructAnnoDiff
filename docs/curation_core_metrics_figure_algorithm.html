核心评估图 A/B/C 算法说明

本页说明 figures/curation_core_metrics_publication.png 的当前统计口径、数据来源和复现方式。

1. 图形目标

这张图用于跨物种概括人工校准前后的三类核心变化： 基因座有无、非 1:1 的拆分/合并事件、以及 严格 1:1 基因中代表性转录本 exon 发生变化的基因比例。

当前图的输入表为 results/curation_core_metrics.csv， 由 generate_tables.py 从每个物种的 results/locus/{species_id}_change_summary.csv 汇总得到。

2. 上游匹配基础

基因模型范围

默认使用 gene_scope=mrna，即只纳入带有 mRNA/transcript 子特征的 gene。 gene span 会扩展到其 mRNA 子特征覆盖范围，用于坐标重叠匹配。

候选重叠

当前批处理默认 overlap_mode=hybrid、 min_reciprocal=0.5。严格 1:1 结构比较使用 reciprocal overlap， split/merge/complex 事件解析使用 containment overlap 构图。

连通分量归类

将 before/after gene 作为二部图节点，候选重叠作为边。每个连通分量按 1:1、1:N、N:1、M:N 分别归为 syntenic、split、merge、complex。

3. Panel A：Locus gain/loss

Panel A 不使用 reciprocal 阈值，也不要求同链；它只回答一个更保守的问题： 一个 before 或 after gene 是否与另一版本中任意 gene 在同一 seqid 上有至少 1 bp 坐标重叠。

图中柱子结果列定义
Deleted loci deleted_loci_no_overlap 校准前 gene 中，与任一校准后 gene 完全没有坐标重叠的基因座数。
New loci new_loci_no_overlap 校准后 gene 中，与任一校准前 gene 完全没有坐标重叠的基因座数。 
new_loci_no_overlap     = count(after_gene with no overlap to any before_gene on same seqid)
deleted_loci_no_overlap = count(before_gene with no overlap to any after_gene on same seqid)

注意：Panel A 是“无坐标重叠基因座”口径，不等同于 change log 中的 strict unmatched after/before gene。

4. Panel B：Split and merge events

Panel B 统计非 1:1 的基因座连通分量事件，属于 gene/locus 层面，不是代表性转录本层面。 因此一个多转录本基因不会因为拥有多个 transcript 而在 Panel B 中重复计数。

图中柱子结果列定义
Split split_events 一个 before gene 与多个 after genes 构成同一连通分量，即 1:N。
Merge merge_events 多个 before genes 与一个 after gene 构成同一连通分量，即 N:1。 
split_events = number of connected components with 1 before gene and N after genes, N >= 2
merge_events = number of connected components with N before genes and 1 after gene, N >= 2

5. Panel C：Representative exon changes

Panel C 只在严格 1:1 gene pair 中判断代表性转录本 exon 是否发生变化。 分子是发生 exon 数量变化或 exon 坐标变化的代表性转录本对数；分母分别换成校准前总 gene 数和校准后总 gene 数， 因此展示的是“这类 1:1 代表性 exon 改变占前后注释规模的比例”。

多转录本处理： 每个 strict 1:1 gene pair 最多贡献一次。before 侧取 primary mRNA，after 侧在该 gene 的所有 mRNA 中选择与 before primary mRNA 最相近的一个。 因此 after 中的多个转录本不会被展开重复计数。

代表性转录本选择

  1. before gene 的代表性转录本为 primary_mrna：优先 CDS 长度最长，其次 exon 数最多。
  2. after gene 的代表性转录本不是简单取第一个，而是在所有 after mRNA 中选择与 before primary mRNA 匹配分数最高者。
  3. 匹配分数优先考虑 exon/CDS/UTR 签名是否一致，然后考虑共享片段数量、CDS 长度差、exon 数差等。

Exon changed 的分子

rep_exon_changed =
    rep_exon_count_changed
  + rep_exon_boundary_changed_same_count
子项定义
rep_exon_count_changed 代表性转录本 before/after exon 数量不同。
rep_exon_boundary_changed_same_count 代表性转录本 exon 数量相同，但 exon 坐标签名 sorted((start, end)) 不同。

Before/After 两个比例

Before ref. = rep_exon_changed / total_before_genes * 100
After ref.  = rep_exon_changed / total_after_genes  * 100

Panel C 当前只展示 exon 变化。CDS 变化子项仍保留在 curation_core_metrics.csv 中用于审计， 但不作为这张图的主面板。
